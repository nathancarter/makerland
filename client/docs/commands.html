Teleport

Leaping around the game world

Memorizing a destination

When you're standing at a location to which you'll want to be able to teleport later, you should take a moment to add it to your list of memorized locations. Here's how:

  1. Click the Teleport command button.
  2. Come up with a name for your current location. This name needs to be meaningful only to you; no one else will see it. Examples, "Black cave" or "by the ocean" or "hiding spot."
  3. Type that name into the box that says "type name here."
  4. Click the Add button.
  5. You should now see the place you just memorized listed among your memorized teleportation destinations, at the top of that same screen.
Warning! You can't teleport somewhere unless you've already memorized that location. So think ahead and memorize an important place before you leave it!
Tip: If you memorize dozens of teleportation destinations, your list will get long and it will be hard to find one when you're looking for it. So choose wisely what you add to your list.
Try it! Walk to one of your favorite locations in the game world and memorize it in that way. Then walk somewhere else, and use the instructions below to teleport back.

Teleporting to a destination

When you want to teleport somewhere, you use the same command. It's very simple:

  1. Click the Teleport command button.
  2. Locate on your list of memorized teleportation destinations the place you want to go.
  3. Click the button next to it that says "Go here."
  4. Your avatar will immediately be transported to that loaction.
Try it! Walk a little distance from one of your memorized teleportation destinations, then use the steps above to teleport back.
Tip: If you click "Go here" and nothing seems to happen, the most likely reason is that you've already teleported to that destination, and you're already there!

Database

Where to store images, sounds, and more

Introduction

If you click the Database command button, you'll see a list of "tables" in the database. Each contains a different kind of information. Here are explanations of each of them, in the order that's easiest to learn (not the order they appear in the game).

  • Abilities - names, descriptions, and code for abilities players can learn as they gain experience
  • Accounts - player accounts, which includes players' names, passwords, and the commands they can use
  • Animations - names, durations, and code for visible animations
  • Behaviors - names, descriptions, and code for behaviors that can be added to items in the world
  • Cell types - each square on the game map is called a "cell," and this table contains the pictures and other data for all the different types of cells there are
  • Creatures - names, images, stats, and behavior for the various people and animals encountered in the game
  • Landscape items - objects that sit on top of the map cells (like trees, rocks, walls, furniture, and so on) are landscape items, and this table contains the pictures and other data for all the different types of landscape items there are
  • Maker error logs - once you start writing code in the game, any errors in it will be stored here for you to browse and fix
  • Movable items - objects that can be picked up and carried around, including food, equipment, and more
  • Planes - each wide, flat area in the game is called a "plane," and makers can create and edit new ones
  • Sounds - sound effects that play when various events occur in the game

Basic tables

A few of the tables in the database are quite straightforward, and you can get begin editing them with just the short introduction given in this section.

Accounts Table

Only the Admin character can change player accounts, so all other makers do not need to bother with this table.

If you are reading this as the Admin character, you can use this table in the database to reset passwords, or to create accounts in a universe in which you have disabled account creation by others.

No instructions are included here about how to use this table, because once you have seen the database interface in the instructions for other tables (below), it is easy to figure out how to use this table. Also, it is infrequently needed.

Planes Table

As a maker, you can create as many new planes in the game as you like, and customize each to your heart's content. For example, you might make one plane that is your own home, and which no one else can visit. Then you might make yet another plane that you want to eventually be an area where players will do a quest to retrieve stolen gold from an evil dragon. You can fill that plane with the map, items, and creatures necessary for that adventure, and when it's all perfect, create a way for (non-maker) players to get there (maybe by boat, or by portal, or through a secret cave).

Try it! To create a new plane, follow the instructions below.
  1. Click the Database command button.
  2. Click the Planes table button.
  3. There may be many planes listed, and most of them will just have a "copy" button. You can always create a new plane by copying an old one, but that's not how we will proceed now. Scroll to the bottom of the list and click "Add entry."
  4. A message will tell you that a new plane has been created. Click OK and scroll to the bottom of the list to find it.
  5. Unlike the other planes on the list, this one will have not only a copy button, but also "remove" and "edit." You could delete it with "remove" if you don't need it any more, but we want this plane, so don't do that. Instead, click "edit."
  6. Before editing any of the plane's details, click the "Teleport to this plane" button near the bottom. Your avatar will immediately appear on an empty surface that looks like graph paper, because it has no cells in it yet. We'll fix that soon.
  7. Click the "Change" button next to your new plane's name. Fill in your own name for it, and click "Change name."
  8. Your name should be the only author listed. No one can edit or remove this plane except you. If you ever want to share authorship, edit that list and add another maker's name. Similarly, if someone shares one of their planes with you, then it will have edit/remove/copy buttons associated with it, rather than just the copy button.
  9. There will be no default cell type for the plane, which is why it looks like graph paper at present. Click the "Change" button for default cell type, and pick a type. If you want a grassy area, choose grass. If you want a desert area, choose sand, and so on.
Tip: Below you will see how to extend this list of cell types with new ones, but for now, choose something already in the database.
Warning! Be sure to choose a cell type that isn't a wall, or water, or something impossible to walk on! You're about to fill your whole plane with this cell type. If you make it something you can't stand on, the game will immediately teleport you out of the plane, until you fix the mistake and teleport back.

That's it! You've created a plane!

Unfortunately, it's all the same repeated pattern everywhere. If you want to make it more interesting, by shaping the land around you with many different cell types and other decorations, move on to the "Changing the world" section, later in this tutorial.

Cell Types Table

This table contains the pictures and other data related to the cells you place on the ground to give a plane its background, as well as dictate where players can walk and where they cannot.

If you click the Cell Types table button in the Database table list, you will see a list of all cell types in the game, with their icons shown. Just as with planes (and most of the database tables) you will see "copy" buttons under most of the cell types, but ones for which you are an author will also have "remove" and "edit" links.

Try it! To create a new cell type, follow the instructions below.
  1. Click the Database command button, then the Cell Types table button.
  2. Just as in the Planes table, above, you may see existing cell types for which you are not the author. Scroll to the bottom and click "Add entry," then find that new entry on the list and click its "edit" button.
  3. The first thing you'll notice is that your new cell type has no icon, which is essential to using it on the map. You will need to upload an image to use as your cell's icon. You can create your own image using an image editing program on your computer, or browse the web for an image that looks the way you want it to. (Just be sure not to take an image unless you have it's author's permission to do so.) When you have an image file you want to use as the icon, proceed to the next step.
  4. Click the Change button next to the icon. Click the Browse button and find on your computer the image file you want to use. Click Start upload, then click OK, and you should see your uploaded icon.
Warning! Using very large image files will slow down the game by creating too much network traffic and too much drawing time in the user's browser. Note the suggested size given to you by the game on the image upload screen, and if your image is too big, first resize it on your computer before uploading it into the game.
  1. Use the various Change buttons to specify the cell type's name and optionally its border size and color.
Tip: Borders are absent by default (size zero) but if you use them, they can help make one particular cell type stand out from its surroundings. Adjacent cell types that both have borders aren't a good idea. You will not see the border until you put some cells onto the game map, as taught below.
  1. If you want the cell type to be walkable, leave the "Who can walk on this?" field set to "all." If you want the cell type to be non-walkable (like a wall, or an ocean) then Change that field and set it to "none."
Sounds Table

This table contains the sound files that are played when various events happen in the game. In the Sample Universe, there are no sound files, and so the game is silent. But it is ready for you to add sound files, and it will play them at appropriate times.

For instance, there is a "login" sound that's played whenever a new player logs in. But you have not yet heard any such sound, because the Sample Universe doesn't have one. Let's add one, as follows.

Try it! To upload a new sound, follow the instructions below.
  1. Click the Database command button, then the Sounds table button.
  2. If you're not in the Sample Universe, you may see existing sounds for which you are not the author. Scroll to the bottom and click "Add entry," then find that new entry on the list and click its "edit" button.
  3. Click the Change button next to the name, and use the name "login" (exactly like that, without quotes, just one single lower-case word). Click Change name.
  4. Next to "Sound file:" it should say [no file], meaning that this sound has not had a file uploaded yet. Click the Change button next to [no file]. Click the Browse button and find on your computer a sound file you want to use. (You may need to record one or to search the web for a suitable sound file.) Click Start upload, then click OK, and you should see that it now reports the size of your uploaded sound file.
Warning! Using very large sound files will slow down the game by creating too much network traffic.
  1. After uploading the sound, Quit the game and log back in. If your computer's sound is turned on, you should hear the login sound played! If not, check to be sure that you named it correctly.

You may naturally wonder what other events in MakerLand will trigger sounds, so that you can upload files for those events as well. Here is the complete list of sound names that the game will trigger by default. Later, when you write your own behaviors, you can have them trigger new sound effects as well.

  • teleport - when the teleportation behavior of a landscape item is triggered, or when a maker uses the teleport command
  • speech bling - when a player speaks (so name because a small bling-like sound is sufficient here)
  • gentle bell - when a player or creature is healed a large amount in a short time (so named as a suggestion of how it could sound, but not a requirement)
  • bone crack - when a player or creature hits another in combat
  • miss - when a player or creature misses another in combat
  • death knell - when a player or creature dies
  • new command access - when a player gets access to a new command for the first time

Intermediate tables

Landscape Items Table

This table contains the pictures and other data related to the items you place on top of the map to give an area its foreground. Such objects have no impact on where players can walk, but they may have behaviors attached to them that take effect when players are nearby.

For example, all the trees, rocks, and potted plants you see around the game world are landscape items. If you've ever stepped on a portal to be transported to a new area, the portal was a landscape item with a teleportation behavior attached to it. And every sign in the game that you can read is a landscape item with a behavior attached to show text when you stand on it.

If you click the Landscape Items table button in the Database table list, you will see a list of all landscape items in the game, with their icons shown. As before, you will see "copy" buttons under most of the entries, but ones for which you are an author will also have "remove" and "edit" links.

Try it! To create a new landscape item, follow the instructions below.
  1. Click the Database command button, then the Landscape Items table button.
  2. Proceed exactly as in the previous two sections to create a new entry and begin editing it. (Scroll down, click "Add entry", click OK, find the entry, click "edit.")
  3. Choose its size (in cells). Most landscape items are the same size as a cell (size 1). All landscape items are square.
  4. Change its icon in exactly the same way that you did for a cell type.
Tip: As with cell types, please use only small files. However, landscape items can have different sizes, so small ones should have small files, and larger ones larger files. The game itself will suggest specific sizes when you choose to upload an icon.
  1. Give the landscape item a name.
  2. Last, you can choose whether the item is visible to players. If you choose "no," then it will be visible only to makers. This is useful for making areas where, if a player steps there, some specific behavior is triggered, but you do not want any visible evidence of this fact. (E.g., some areas do something when you enter them, such as play a sound or bring a greeter; other places you may wish to position a trap.)

You have created a new landscape item, but not yet placed any copies of it in the game world. See the section below on the "World" command to learn how!

Movable Items Table

This table contains the pictures and other data related to the items carried by players and creatures. This includes things like food, equipment, treasure, books, and more. Such things may have behaviors attached to them; for instance, a sword has the behavior that when you equip it, you gain an increase in your attacking ability. A book has the behavior that when you look in your inventory, it will let you read it.

If you click the Movable Items table button in the Database table list, you will see a list of all movable items in the game, with their icons shown. As before, you will see "copy" buttons under most of the entries, but ones for which you are an author will also have "remove" and "edit" links.

Try it! To create a new movable item, follow the instructions below.
  1. Click the Database command button, then the Movable Items table button.
  2. Proceed as above to create a new entry and begin editing it. (Scroll down, click "Add entry", click OK, find the entry, click "edit.")
  3. Choose the number of spaces it takes up in a player's inventory. This should be a whole number. Players can carry 10 items by default. Small items usually have a size of 1, and something should only have a size of 10 if it is so large that the player can carry nothing else. Reasonable sizes for weapons and armor include 2, 3, or 4 for very bulky items.
  4. Change its icon in exactly the same way that you did for a cell type.
Tip: As above, please use only small files. Movable items should all be rather small, since it is unrealistic to be able to carry around something very large.
  1. Give the movable item a name.

You have created a new movable item, but not yet placed any copies of it in the game world. To do so, you need to use a behavior, covered below. For instance, some creatures have the behavior that they always spawn carrying an item, which players can obtain by killing the creature. Other creatures will trade one item for another with players. In these ways (and others you may invent), players can obtain movable items.

Creatures Table

This table contains the pictures and other data related to the creatures you encounter in the game. This includes both friendly creatures and hostile ones.

If you click the Creatures table button in the Database table list, you will see a list of all creatures in the game, with their icons shown. As before, you will see "copy" buttons under most of the entries, but ones for which you are an author will also have "remove" and "edit" links.

Try it! To create a new creature, follow the instructions below.
  1. Click the Database command button, then the Creatures table button.
  2. Proceed exactly as in the previous two sections to create a new entry and begin editing it. (Scroll down, click "Add entry", click OK, find the entry, click "edit.")
  3. Change its icon in exactly the same way that you did for a cell type.
Tip: As with cell types, please use only small files. However, creatures can have different sizes, so small ones should have small files, and larger ones larger files. The game itself will suggest specific sizes when you choose to upload an icon.
  1. Give the creature a name.
  2. The remainder of the creature's attributes are related to how it performs in combat. The default values for these statistics are the same as they are for players who have just started playing the game and have not yet earned any kind of bonuses or increases. Change them as you see fit. To know what impact "dodging ability" or "attack accuracy" has at various values, create such creatures and experiment with them in combat!
  3. "Experience" is the amount of experience earned for killing the creature, earned by the player who deals the final blow. When you click Change, the game performs a very rudimentary computation to suggest a reasonable value based on the creature's other statistics. Feel free to use it, or adjust it.

You have created a new creature, but not yet placed any copies of it in the game world. To do so, you will need to use a behavior, covered below. For isntance, a landscape item may have the behavior that every so often, it spawns a creature.

Advanced tables

Abilities Table

This table contains the descriptions, icons, and code for abilities players can earn as they journey through your universe. These may be abilities for use in combat, navigation, communication, or anything you create.

Abilities show up as new commands in the panel on the right hand side of the player's screen.

⏩ Read about the Abilities table on the Coding page.

Animations Table

This table contains the code for various animated effects that appear in the world, such as the text that appears when you read a sign, speech bubbles that appear when you talk, and the lunging motions of your character during combat.

Animations are written in JavaScript and stored in this table, to be played on screen when events in the game require them.

⏩ Read about the Animations table on the Coding page.

Behaviors Table

This table contains the descriptions, parameters, and code for behaviors makers can attach to various things in the universe. Examples:

  • To make a sign show text when you step on it, you attach that ability to the sign, and specify the text as a parameter.
  • To make a location in the game world spawn a creature, you attach the spawning ability to a landscape item, and identify the creature in a parameter.
  • To make a creature wander after it spawns, you attach the wandering behavior to the creature, and specify the rate and other parameters.

⏩ Read about the Behaviors table on the Coding page.

Maker Error Logs Table

When you code, inevitably, sooner or later, errors arise. You will want reports on those errors, so that you can fix them. The Maker Error Log is where errors are stored for later reading by the maker who wrote the code that generated the error.

⏩ Read about the Maker Error Log table on the Reference page.

World

Changing the things around you

Warning! This command functions only in planes for which you are the author. If you're in a common area, it's probably authored by the Admin character, and unles you're the Admin, you can't edit it. If you're in someone else's plane, they're probably the only author, and you can't edit it. To learn more about authorship for a plane, read about the Database command. For now, the easiest way to use this command is to make your own plane (as described at that same location) and then use the World command in that plane.

Changing the cells of the map

The base layer of the map, underneath any trees, rocks, plants, signs, pillars, and so on, is a grid of squares, called cells.

The first thing you'll want to edit in a brand new plane you've created is that grid of cells. Here's how.

  1. As described above, be sure you're in a plane that you have permission to edit.
  2. Click the World command button.
  3. Click "Change cells in map."
  4. Click the "Change" button to choose which kind of cells you want to be placing on the map. Scroll down the list until you find the one you want to use, then click its "Choose" button.
  5. Click "Place individual cells".
  6. When you aim the mouse over the map itself, you should now see a crosshair cursor (like a plus), indicating that you're able to edit the map. Anywhere you click, you will place a copy of your chosen cell.
  7. When you're done placing that type of cell, click "Done."
  8. You can then pick another type of cell, or click "Done" if you're finished editing the map cells.
Tip: If you don't see the type of cells you want, return to the page on the database command and follow its directions to add new cell types to the game!
Try it! Enjoy some time editing your own private part of the game world! Shape it to your own desires!
Tip: You probably noticed two other buttons in the "Editing map cells" screen. Each is there to make it easier to edit large areas of cells all at once. They function as follows:
  • Fill rectangles of cells - after choosing this option, click the top left cell of the area you want to fill, then the bottom right. The entire rectangular area between your two click points will be filled with the same type of cell.
  • Rectangular border of cells - same as the previous, but does not fill the interior of the rectangular region, only its outer border (one cell thick)

Decorating with items

The upper layer of the game map consists of objects that sit on top of the map cells. These objects need not be one-cell-by-one-cell in size, and they need not be images that fill a square. They're called "landscape items." Examples include plants, rocks, furniture, signs, and more.

Here's how to add landscape items to your plane.

  1. As described above, be sure you're in a plane that you have permission to edit.
  2. Click the World command button.
  3. Click "Put landscape items on map."
  4. As you did when placing cells on the map, begin by choosing a landscape item to place on the map. Click "Change," then find one you want to use, then click "Choose."
  5. Click "Place landscape items".
  6. When you aim the mouse over the map itself, you should now see a crosshair cursor (like a plus), indicating that you're able to edit the map. Anywhere you click, you will place a copy of your chosen landscape item. The item will be centered on your click point.
  7. When you're done placing that type of item, click "Done."
  8. You can then pick another type of landscape item, or click "Done" if you're finished adding landscape items to the map.
Tip: If you don't see the type of cells you want, return to the page on the database command and follow its directions to add new cell types to the game!
Try it! Enjoy some time decorating your part of the game world!
Tip: You can also edit landscape items once you've placed them. The most common use of this is if you placed something in the wrong location. Choose "Edit landscape items on map," then click on an item on the map. Assuming you clicked on just one item, you will then have the option to delete it or edit its behaviors (an advanced topic). If you clicked a spot where more than one item is stacked on top of one another, then the game will prompt you with a list, from which you can choose the item you meant.

Reset

Cleaning up your surroundings

The game map is split up into rectangular blocks of cells. As players walk onto new areas of the map, the blocks of cells for those new areas are loaded from disk so that they are ready to be shown to the player. Similarly, the blocks that the player can no longer see are dropped from the game's memory, since no player needs them.

Each block contains not only cells (the square images that make the map look how it does), but also the landscape items that sit on top of the cells (e.g., trees, rocks, pillars, signs, etc.). When a block of cells is loaded from disk into memory, all its landscape items are also loaded, and the behavior code for each is run.

If you've filled your environment with objects you no longer want, such as items you created into your inventory and then dropped, or creatures you manually spawned near you, you may wish to clean it up. Issue the reset command to destroy all such temporary items.

Warning! The Reset command only makes sense after you've learned how to code a behavior or an animation. If you haven't learned those things yet, then you don't need the Reset command.
Tip: Although it's possible to accomplish this by walking very far away from your current location, then walking back, that's a waste of time, and it only works if no one else is around.
Try it! To reset the area you can see, click the Reset command button. You will not necessarily see anything happen on the game map, but if you edited anything about landscape items (such as their behaviors) your changes have now been applied.
