Getting Started

Learning to be a Maker

This hands-on guide teaches you how to change the world around you in MakerLand. From the first steps, you'll gain new powers and see how to put them into action.

It starts easy, with topics that require no coding, no art, and no planning ahead. More advanced topics appear later.

By the end, you will be able to fully sculpt the Sample Universe, or some other universe, or make a universe from scratch!

Teleporting

You can choose a set of locations in the game world that are important to you, and then later instantaneously leap to any one of them.

⏩ Read about the Teleport command on the Commands page.

⏪ Come back here afterward. (Completed the training, you have not.)

Adding to the Game

The Game Database is the list of all things the game knows about, including player accounts, square images that make up the map, other images that sit on top of the map, sounds, animations, behaviors, and more. The Database command lets you add to this big library of information, so that you can put new things into the game.

⏩ Read about the Database command on the Commands page. For now, just read the the Introduction and Basic Tables sections. (We will do the more advanced sections later.)

⏪ Come back here afterward.

Warning! The Database command does not allow you to edit the world around you. The Game Database is just the collection of things that could be added to the game world. After you learn how to add things to the database, we'll look at the World command, which lets you then place copies of them all around the game world. (That's the section after this one.)

Getting interactive

All the changes you've made to your environment so far are either non-interactive (such as changing the map or the items that sit, unmoving, on top of it) or require your presence (such as creating new creatures in your environment). Of course, you will want to know how to make interactive places that function even when you are not standing there executing maker commands! That's where behaviors come in.

⏩ Visit the Coding page, but just read the first three sections about Behaviors. (That is, read from "What are behaviors?" through "Many opportunities." These sections do not require you to learn to code.)

⏪ Come back here afterward to try out your most interesting projects yet!

The expanding universe

There are a LOT of things you can now do with the Sample Universe, leveraging what you know about behaviors. This is a great place to stop learning new things and just practice what you already know.

⏩ Visit the Ideas page, and read the section on Basic Behaviors. Pick and choose whatever seems most interesting to you, and do it in your copy of the Sample Universe.

⏩ Read the Writing Behaviors section of the Coding page, which includes LOTS of reference code to learn from and ideas you can try to build as you learn.

⏪ Come back here when you're done building and want to move beyond behaviors, to animations and abilities.

Adding animation

If you've been playing in the Sample Universe, you may notice that it's lacking a lot in the animation department. Things are pretty rudimentary. But that's not because the game has no animations. Rather, it is because those animations are fully customizable, and the Sample Universe comes with only bare bones customization regarding animations.

You can add animations to the game that happen in response to game events such as speaking, combat, or movement, or that happen whenever you want them to, based on behaviors you've written.

⏩ Read the Animations section of the Coding page, learn from the built-in example animations in the Sample Universe, and try some of the animation ideas on the coding page.

⏪ Come back here when you want to move to the final coding lesson, on abilities.
